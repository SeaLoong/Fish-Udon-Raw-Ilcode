## 1. 项目概述

本仓库包含了 VRChat 世界 **Fish World**（一个大型多人在线钓鱼模拟游戏）中 Udon 程序的反编译 IL 字节码，以及由分析工具自动生成的元数据索引。当前快照包含 **172 个程序**。

Fish World 是一个功能极为完善的 VRChat 世界，涵盖：

- 完整的**钓鱼模拟**（鱼竿物理、抛竿、钓鱼小游戏、鱼类生态）
- **RPG 式角色系统**（等级、经验、金币、成就、称号）
- **装备系统**（鱼竿、鱼线、浮漂，各带属性与附魔）
- **船只驾驶**（物理漂移、加速器、皮肤自定义）
- **宠物系统**（AFK 自动钓鱼、升级）
- **动态世界**（天气系统、日夜循环、区域音乐）
- **经济系统**（商店、悬赏、每日登录、兑换码）
- **AES-256 GPU 加密**（Discord 权限验证）

## 3. 元数据文件说明

### programnames（程序名称映射）

将 VRChat 世界中的 **GameObject 名称**映射到其使用的**程序哈希**：

```text
Siamese Giant Carp 1 : 02cca     # 暹罗巨鲤 → PlayerFishPickup 脚本
Lil Yachty(Clone) : 292a2        # 小游艇 → 船只控制脚本
Player Inventory Data : 04a6d    # 玩家背包数据脚本
AudioMasterScript : 573f4        # 音频主控脚本
```

同一个程序哈希被大量 GameObject 复用（例如 `02cca` 被所有鱼类对象使用）。

### functiondb（函数数据库）

JSON 格式存储每个程序的所有函数签名：

- `start`/`end`：IL 字节码偏移量
- `name`：内部函数名
- `publicname`：公开函数名
- `accessibility`：访问修饰符
- `returntype`：返回类型

### dirlist（函数目录索引）

扁平化的函数列表，用于快速查找：

```text
292a2 : [0, get_OwnerPlayerId]
292a2 : [15768, _start]
292a2 : [19296, _update]
292a2 : [47172, _onCollisionEnter]
```

### varlist（变量列表）

详细记录每个变量的类型、默认值、读写位置、跨程序引用：

```text
02cca : __refl_typename/0/0 = "PlayerFishPickup"
02cca : InitialScale/2/0 = new Vector3(0f, 0f, 0f)
```

### declibrary（声明库）

巨大的 JSON 文件（约 591K 行），包含 VRChat Udon 虚拟机可用的**全部 32,425 个外部 API 声明**，涵盖 Unity 引擎 API、VRChat SDK API 等。

### stats（统计信息）

仓库包含两次反编译运行的数据，统计差异如下：

| 指标       | 值         |
| ---------- | ---------- |
| 程序总数   | 172        |
| 对象引用数 | 4,359      |
| baseobject | 29.82 MB   |
| functiondb | 710.30 KB  |
| declibrary | 22.42 MB   |

## 4. 程序目录结构

每个程序以其 **MD5 哈希**的前 5 位作为简称标识。每个目录下包含：

| 文件            | 内容         | 用途                                   |
| --------------- | ------------ | -------------------------------------- |
| `altopcode`     | 高级伪代码   | **最适合人类阅读**，带控制流、条件分支 |
| `exactexec`     | 精确执行序列 | 逐条指令的执行记录                     |
| `opcode`        | 原始操作码   | Udon VM 的原始指令                     |
| `program64`     | Base64 编码  | 原始二进制程序数据                     |
| `variables64_*` | Base64 变量  | 原始变量区数据                         |
| `variablesjs_*` | JSON 变量    | 结构化变量信息                         |

## 6. AES-256-CBC GPU 加密实现深度分析

这是本项目中技术含量最高的部分——在 VRChat Udon 严格沙箱限制下，完整实现了 **AES-256-CBC 解密**。

### 6.1 背景与动机

VRChat 的 Udon 运行时禁止使用 .NET 的 `System.Security.Cryptography` 命名空间。为了安全地从远程服务器传输 Discord 角色/权限数据，开发者**从零实现了整套 AES-256 解密**，并巧妙地将最耗时的部分卸载到 **GPU Shader** 上。

### 6.2 架构概览

```text
[远程服务器] ──Base64(IV + AES-256-CBC(JSON))──► [VRCStringDownload]
                                                        │
                                        Base64 解码 → 分离 IV (16B) + 密文
                                                        │
                                        密钥扩展 (CPU, AES-256 KeySchedule)
                                                        │
                                  ┌─────────────────────┴──────────────────────┐
                                  │  GPU ECB 解密 (Shader + Texture2D + Blit)  │
                                  │  密文块 → 像素数据 → GPU Shader → 异步回读  │
                                  └─────────────────────┬──────────────────────┘
                                                        │
                                        CBC XOR 链还原 (CPU, 分帧)
                                                        │
                                        PKCS7 去填充 → UTF-8 字符串
                                                        │
                                  JSON 解析 → Discord 角色与权限验证
```

### 6.3 核心流程分解

#### 阶段 1：密钥准备（`_start`）

启动时，`DiscordRoleManager._start()` 从堆常量构建**混淆的口令和盐值字节数组**（各 32 字节），通过 XOR 解混淆解码为 ASCII 字符串，再经 UTF-8 编码后传入 `CryptoAES256GPU.DeriveKey()` 派生出 32 字节 AES 密钥。

**口令和盐值使用独立的混淆方案**：
- 口令: `decoded[i] = (raw[i] ^ k[i%7]) ^ ((i*15+69) & 255)`（使用 7 字节密钥因子 `k`）
- 盐值: `decoded[i] = (raw[i] ^ 96) ^ (i & 31)`

`CryptoAES256GPU._start()` 初始化过程中构建：

- **AES S-Box**（256 字节正向替换表）
- **逆 S-Box**（`invSBox`，256 字节逆向替换表）
- **Rcon 常量表**（10 个轮常量，用于密钥扩展）

详细的密钥派生分析见 [AES-256 IL 级分析文档](aes256-il-analysis-zh.md)。

#### 阶段 2：数据获取（`_FetchRoleData` → `_onStringLoadSuccess`）

使用 VRChat 的 `VRCStringDownload` 从远程 URL 下载 Base64 编码的加密字符串。若 `useEncryption = true`，进入解密流程。

#### 阶段 3：Base64 解码与 IV 分离（`__0_DecryptStringGPU`）

- `FromBase64String()` 解码
- 前 16 字节提取为 **IV**（初始化向量）
- 剩余部分为 **密文**（`_ciphertext`）

#### 阶段 4：密钥扩展（`__0_ExpandKey`）

标准 AES-256 密钥扩展算法：

- 从 32 字节密钥生成 **240 字节扩展密钥**
- Nk=8（256 位密钥 = 8 个 32 位字），Nr=14（14 轮）
- 使用 S-Box 进行 SubWord、RotWord，与 Rcon 异或

#### 阶段 5：GPU ECB 解密（分块处理）

这是最巧妙的部分：

1. 将密文块和扩展密钥**打包为 `Texture2D` 像素数据**（`Color32` 数组）
2. 将纹理传入 `computeMaterial`（一个专门编写的 GPU Shader）
3. 使用 `Graphics.Blit()` 在 GPU 上执行 **AES ECB 单块解密**
   - Shader 内部实现了完整的 AES 逆操作：InvSubBytes / InvShiftRows / InvMixColumns / AddRoundKey
4. 通过 `AsyncGPUReadback` 异步回读结果
5. 分块处理（`_currentChunkIndex`），每次处理固定数量的 16 字节块，避免单帧卡死

#### 阶段 6：CBC 模式还原（`_ContinueCBC`）

GPU 仅做 ECB 解密（无状态、可并行）。CBC 的异或链在 CPU 端完成：

```text
plaintext[i] = ecbDecrypted[i] XOR previous_ciphertext_block[i]
plaintext[0] = ecbDecrypted[0] XOR IV
```

这一步也采用分帧处理（每帧最多 258 个块），通过 `SendCustomEventDelayedFrames` 延续。

#### 阶段 7：PKCS7 去填充与结果返回

- 检查最后一个字节确定填充长度
- 验证填充有效性
- `UTF8.GetString()` 转为字符串
- 通过回调返回给 `DiscordRoleManager`

### 6.4 密钥派生函数（`__0_DeriveKey`）

系统通过自定义 KDF（口令 + Salt）派生 AES-256 密钥：

1. 口令字符串经 `UTF8.GetBytes()` 编码为字节数组
2. 与 Salt 字节拼接为 `combined` 数组
3. 循环填充初始化 32 字节 `derived` 数组：`derived[i] = combined[i % combined.length]`
4. 执行 **1000 轮**混合迭代：

   ```text
   mixed = prev ^ curr ^ next ^ salt[(round+j) % salt.length] ^ (round % 256)
   temp[j] = sBox[mixed & 0xFF]
   ```

5. 输出最终 32 字节派生密钥

口令和 Salt 均为 32 个 XOR 混淆字节，解混淆后为 ASCII 字符串。完整 IL 跟踪见 [AES-256 IL 级分析文档](aes256-il-analysis-zh.md)。

### 6.5 技术意义

这是一个极具创造力的实现——在 Udon 这种严重受限的沙箱环境中：

- **绕过加密库限制**：完全手写 AES 算法
- **GPU 加速**：利用 Shader 替代 CPU 密集计算
- **分帧处理**：避免帧率卡顿，保持 VR 体验流畅
- **完整 CBC 模式**：CPU 端完成 XOR 链，实现工业级加密标准
