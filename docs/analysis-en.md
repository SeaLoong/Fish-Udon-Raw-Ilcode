## 1. Project Overview

This repository contains the **decompiled IL bytecode of all 174 Udon programs** from the VRChat world **Fish World** — a fully-featured multiplayer online fishing simulation game — along with auto-generated metadata indices produced by the analysis tooling.

Fish World is an exceptionally comprehensive VRChat world featuring:

- Full **fishing simulation** (rod physics, casting, fishing minigame, fish ecology)
- **RPG-style character system** (levels, XP, currency, achievements, titles)
- **Equipment system** (rods, lines, bobbers, each with stats and enchantments)
- **Boat driving** (physics-based drifting, boost, skin customization)
- **Pet system** (AFK auto-fishing, upgradeable stats)
- **Dynamic world** (weather system, day/night cycle, zone-based music)
- **Economy** (shops, bounties, daily login, redeem codes)
- **AES-256 GPU encryption** (Discord role verification)

## 3. Metadata Files

### programnames (Program Name Mappings)

Maps VRChat world **GameObject names** to their **program hashes**:

```
Siamese Giant Carp 1 : 02cca     # Fish object → PlayerFishPickup script
Lil Yachty(Clone) : 292a2        # Small yacht → Boat controller script
Player Inventory Data : 04a6d    # Player inventory data script
AudioMasterScript : 573f4        # Audio master controller
```

The same program hash is reused across many GameObjects (e.g., `02cca` is used by all fish objects).

### functiondb (Function Database)

JSON file storing all function signatures for every program:

- `start`/`end`: IL bytecode offsets
- `name`: internal function name
- `publicname`: public function name
- `accessibility`: access modifier
- `returntype`: return type

### dirlist (Function Directory Index)

Flattened function listing for quick lookups:

```
292a2 : [0, get_OwnerPlayerId]
292a2 : [15768, _start]
292a2 : [19296, _update]
292a2 : [47172, _onCollisionEnter]
```

### varlist (Variable List)

Detailed record of every variable's type, default value, read/write locations, and cross-program references:

```
02cca : __refl_typename/0/0 = "PlayerFishPickup"
02cca : InitialScale/2/0 = new Vector3(0f, 0f, 0f)
```

### declibrary (Declaration Library)

A massive JSON file (~591K lines) containing all **32,425 external API declarations** available to the Udon VM, covering Unity Engine APIs, VRChat SDK APIs, and more.

### stats (Statistics)

```
warmup:7679          # Warm-up time (ms)
worlddata:1267       # World data processing
opcodealt:902        # High-level pseudocode generation
execlog:569          # Execution log generation
totaltime:47015      # Total processing time (ms)
```

## 6. AES-256-CBC GPU Encryption — Deep Dive

This is the most technically impressive component — a complete **AES-256-CBC decryption** implementation within VRChat's severely restricted Udon sandbox.

### 6.1 Background & Motivation

VRChat's Udon runtime blocks access to .NET's `System.Security.Cryptography` namespace. To securely transmit Discord role/permission data from a remote server, the developers **implemented AES-256 decryption from scratch** and cleverly offloaded the most compute-intensive parts to the **GPU via Shaders**.

### 6.2 Architecture Overview

```
[Remote Server] ──Base64(IV + AES-256-CBC(JSON))──► [VRCStringDownload]
                                                          │
                                          Base64 Decode → Split IV (16B) + Ciphertext
                                                          │
                                          Key Expansion (CPU, AES-256 KeySchedule)
                                                          │
                                    ┌─────────────────────┴──────────────────────┐
                                    │  GPU ECB Decrypt (Shader + Texture2D + Blit)│
                                    │  CT blocks → pixels → GPU Shader → readback │
                                    └─────────────────────┬──────────────────────┘
                                                          │
                                          CBC XOR Chain (CPU, frame-split)
                                                          │
                                          PKCS7 Unpadding → UTF-8 String
                                                          │
                                    JSON Parse → Discord Role & Permission Verification
```

### 6.3 Core Flow Breakdown

#### Phase 1: Key Preparation (`_start`)

At startup, the 32-byte AES-256 key (hardcoded array) is passed to `CryptoAES256GPU` via `SetProgramVariable`, triggering initialization. The initialization builds:

- **AES S-Box** (256-byte forward substitution table)
- **Inverse S-Box** (`invSBox`, 256-byte reverse substitution table)
- **Rcon constant table** (10 round constants for key expansion)

#### Phase 2: Data Retrieval (`_FetchRoleData` → `_onStringLoadSuccess`)

Uses VRChat's `VRCStringDownload` to download the Base64-encoded encrypted string from a remote URL. If `useEncryption = true`, enters the decryption pipeline.

#### Phase 3: Base64 Decode & IV Extraction (`__0_DecryptStringGPU`)

- `FromBase64String()` decodes the input
- First 16 bytes extracted as the **IV** (Initialization Vector)
- Remaining bytes become the **ciphertext** (`_ciphertext`)

#### Phase 4: Key Expansion (`__0_ExpandKey`)

Standard AES-256 key expansion:

- Generates **240-byte expanded key** from the 32-byte key
- Nk=8 (256-bit key = 8 × 32-bit words), Nr=14 (14 rounds)
- Uses S-Box for SubWord, RotWord, XOR with Rcon

#### Phase 5: GPU ECB Decryption (Chunked)

This is the most ingenious part:

1. Packs ciphertext blocks and expanded key into **`Texture2D` pixel data** (`Color32` arrays)
2. Passes textures to `computeMaterial` (a custom GPU Shader)
3. Uses `Graphics.Blit()` to execute **AES ECB single-block decryption** on the GPU
   - Shader implements complete AES inverse operations: InvSubBytes / InvShiftRows / InvMixColumns / AddRoundKey
4. Uses `AsyncGPUReadback` for asynchronous result readback
5. Processes in chunks (`_currentChunkIndex`), handling a fixed number of 16-byte blocks per pass to avoid frame stalls

#### Phase 6: CBC Mode Reconstruction (`_ContinueCBC`)

The GPU handles only ECB decryption (stateless, parallelizable). The CBC XOR chain is completed on the CPU:

```
plaintext[i] = ecbDecrypted[i] XOR previous_ciphertext_block[i]
plaintext[0] = ecbDecrypted[0] XOR IV
```

This step also uses frame-split processing (max 258 blocks per frame) via `SendCustomEventDelayedFrames`.

#### Phase 7: PKCS7 Unpadding & Result Return

- Reads the last byte to determine padding length
- Validates padding correctness
- Converts to string via `UTF8.GetString()`
- Returns to `DiscordRoleManager` via callback

### 6.4 Key Derivation Function (`__0_DeriveKey`)

In addition to hardcoded keys, the system supports passphrase + salt key derivation:

1. Concatenate passphrase UTF-8 bytes with salt
2. Cycle-fill into a 32-byte buffer
3. Apply multiple mixing rounds: `derived[i] = prev ^ SBox(curr) ^ next ^ saltByte`
4. Output 32-byte key

### 6.5 Technical Significance

This is a remarkably creative implementation within Udon's heavily constrained sandbox:

- **Bypasses crypto library restrictions**: hand-written AES algorithm from scratch
- **GPU acceleration**: uses Shaders instead of CPU-intensive computation
- **Frame-split processing**: avoids frame drops, maintaining smooth VR experience
- **Full CBC mode**: CPU-side XOR chain completes industry-standard encryption
